Fixed hard maximum number of pending asynchronous queue entries per file sink.

This is a non-configurable safety limit, not the queue size itself. The async queue preallocates every slot, so an oversized `queue_capacity` can panic the process at startup; configuration above this bound is rejected with a config error. It cannot be raised.
